MediaWiki.api module¶

class MediaWiki.api.API(api_url, **kwargs)¶

Bases: MediaWiki.connection.Connection

Simple interface to MediaWiki’s API.

This object should implement wrappers around the ‘action’ parameter, the “public” methods are named as its possible values (login, query, edit, ...) and helper methods start with an underscore (‘_’).

Parameters:
  • api_url – URL path to wiki API interface
  • kwargs – any keyword arguments of the Connection object
edit(pageid, text, basetimestamp, summary, token=None, **kwargs)¶

Interface to API:Edit.

Parameters:
  • pageid – page ID of the page to be edited
  • text – new page content
  • basetimestamp – Timestamp of the base revision (obtained through prop=revisions&rvprop=timestamp). Used to detect edit conflicts.
  • summary – edit summary
  • kwargs – Additional query parameters, see API:Edit.

MediaWiki.connection module¶

class MediaWiki.connection.Connection(api_url, cookie_file=None, cookiejar=None, user_agent='wiki-scripts/0.4 (https://github.com/lahwaacz/wiki-scripts)', http_user=None, http_password=None, ssl_verify=None)¶

Bases: builtins.object

The base object handling connection between a wiki and scripts.

It is possible to save the session data by specifying either cookiejar or cookie_file arguments. This way cookies can be saved permanently to the disk or shared between multiple Connection objects. If cookiejar is present cookie_file is ignored.

Parameters:
  • api_url – URL path to the wiki API endpoint
  • cookie_file – path to a cookielib.FileCookieJar file
  • cookiejar – an existing cookielib.CookieJar object
  • user_agent – string sent as User-Agent header to the web server
  • ssl_verify – if True, the SSL certificate will be verified
call(params=None, expand_result=True, **kwargs)¶

Convenient method to call the API.

Checks the action parameter (default is "help" as in the API), selects correct HTTP request method, handles API errors and warnings.

Parameters of the call can be passed either as a dict to params, or as keyword arguments.

Parameters:
  • params – dictionary of API parameters
  • expand_result – if True, return only part of the response relevant to the given action, otherwise full response is returned
  • kwargs – API parameters passed as keyword arguments
Returns:

dictionary containing (part of) the API response

diff(oldrevid, newrevid)¶

Method to get highlighted diff of two revisions. Uses ANSI color sequences for output in a 256-color terminal.

Basic meta data (title, username, timestamp and comment) is included in the diff header. Original unified diff format supports only file name and timestamp fields, we show more.

Parameters:
  • oldrevid – revision ID for old revision
  • newrevid – revision ID for new revision
MediaWiki.diff.diff_highlighted(old, new, fromfile='', tofile='', fromfiledate='', tofiledate='')¶

Generic wrapper around difflib.unified_diff() with highlighter based on pygments.

See difflib.unified_diff for description of optional parameters.

Parameters:
  • old – text to compare (old revision)
  • new – text to compare (new revision)
  • fromfile – original file name (used as meta data to format diff header)
  • tofile – new file name (used as meta data to format diff header)
  • fromfiledate – original file timestamp (used as meta data to format diff header)
  • tofiledate – new file timestamp (used as meta data to format diff header)
